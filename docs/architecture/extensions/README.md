# Extension Recipes

Extension recipes are derivation templates for adding new features after a subsystem has stabilized. Individual recipes land at the tail of their owning phase; this README and template define the framework.

| Feature | Recipe |
|---|---|
| Add AI behavior/creature/action | `ai.md` (phase-tail recipe) |
| Add state slice/action/reducer | `state.md` (phase-tail recipe) |
| Add vCam effect/state | `vcam.md` (phase-tail recipe) |
| Add ECS component/system/event | `ecs.md` (phase-tail recipe) |
| Add manager | `managers.md` (phase-tail recipe) |
| Add UI screen/overlay/panel | `ui.md` (phase-tail recipe) |
| Add scene/transition/registry entry | `scenes.md` (phase-tail recipe) |
| Add save field/migration | `save.md` (phase-tail recipe) |
| Add input action/profile | `input.md` (phase-tail recipe) |
| Add audio track/event | `audio.md` (phase-tail recipe) |
| Add objective/victory route | `objectives.md` (phase-tail recipe) |
| Add condition/effect/rule | `conditions_effects_rules.md` (phase-tail recipe) |
| Add event type/subscription | `events.md` (phase-tail recipe) |
| Add debug/perf surface | `debug.md` (phase-tail recipe) |
| Add display/post-process preset | `display_post_process.md` (phase-tail recipe) |
| Add localization key/language | `localization.md` (phase-tail recipe) |
| Add validated resource | `resources.md` (phase-tail recipe) |
| Add test/style guard | `tests.md` (phase-tail recipe) |

Use [TEMPLATE.md](TEMPLATE.md) when authoring a new recipe.
