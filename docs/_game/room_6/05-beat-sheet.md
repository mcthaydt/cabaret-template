## Beat ID scheme

Numbered `B001`–`B056`. If a beat needs to split during implementation, use `B017a` / `B017b` and note the fork in **Next-beat gate**. Do not renumber — beat IDs must be stable addresses.

## Authoring invariants (do not violate during implementation)

- **No silent flag sets.** Every row that sets a flag must have a gated beat downstream that *reads* it, or the flag is dead weight.
- **Gate determinism.** No beat advances on time-alone unless the row explicitly says "Xs elapsed" in **Next-beat gate**. Player-driven gates are preferred wherever plausible.
- **Camera always named.** No beat has a blank Camera/vcam cell — use `free clamp` explicitly if the player has free look.
- **Cross-refs resolved.** Every scene/vcam/item/flag/cutscene ID in this table exists in docs 02/03/04.
